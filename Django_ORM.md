# Django Technical Paper

## Introduction
This document explains Django ORM, how to use it in Django shell, converting ORM queries to SQL, aggregations, annotations, migrations, and SQL transaction concepts.

## 1. Django ORM

Django ORM (Object Relational Mapping) is a feature that allows developers to interact with the database using Python code instead of SQL.

### Key Features
- Converts Python code into SQL queries
- Database independent
- Easy CRUD operations
- Improves productivity

### Example
    from app.models import Student
    students = Student.objects.all()

## 2. Using ORM Queries in Django Shell

Django shell allows you to interact with your project using Python.

### Start Shell
    python manage.py shell

### Common Queries
    Student.objects.all()
    Student.objects.filter(age=20)
    Student.objects.get(id=1)
    Student.objects.create(name="John", age=22)

### Operations
- Create → create()
- Read → all(), filter(), get()
- Update → save()
- Delete → delete()

## 3. Turning ORM to SQL in Django Shell

You can view the SQL query generated by ORM.

### Example
    queryset = Student.objects.filter(age=20)
    print(queryset.query)

### Purpose
- Debugging
- Understanding SQL behind ORM
- Performance optimization

## 4. What are Aggregations?

Aggregations are used to perform calculations on a set of data and return a single value.

### Examples
- Count
- Sum
- Avg
- Max
- Min

### Example
    from django.db.models import Avg
    Student.objects.aggregate(Avg('age'))

### Use Case
- Calculate average, total, or count of records

## 5. What are Annotations?

Annotations are used to add calculated fields to each object in a queryset.

### Difference from Aggregation
- Aggregation → Returns a single result
- Annotation → Adds value to each row

### Example
    from django.db.models import Count
    Course.objects.annotate(student_count=Count('student'))

### Use Case
- Add extra computed data to query results

## 6. What is a Migration File?

A migration file records changes made to models.

### Purpose
- Apply database schema changes
- Keep database in sync with models

### Commands
    python manage.py makemigrations
    python manage.py migrate

### Key Points
- Stored in migrations folder
- Version control for database
- Tracks changes over time

## 7. What are SQL Transactions? (Non-ORM Concept)

A SQL transaction is a sequence of operations performed as a single unit of work.

### Properties (ACID)
- Atomicity → All or nothing
- Consistency → Valid state
- Isolation → Independent execution
- Durability → Permanent changes

### Example
    BEGIN;
    UPDATE account SET balance = balance - 100 WHERE id = 1;
    UPDATE account SET balance = balance + 100 WHERE id = 2;
    COMMIT;

### Purpose
- Ensure data integrity
- Prevent partial updates

## 8. What are Atomic Transactions?

Atomic transactions ensure that a block of operations is executed completely or not at all.

### Django Support
Django provides atomic transactions using transaction.atomic()

### Example
    from django.db import transaction

    with transaction.atomic():
        # multiple database operations
        pass
